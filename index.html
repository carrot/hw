About

Hemingway is a tool to easily manage, run, and collaborate on miscellaneous tasks.

Installation

$ gem install hw
$ hw update

Usage

Updating sources

$ hw update

Viewing all available packages

$ hw list

Viewing all available tasks in a given package

$ hw [package name]
$ hw airbrake

Running a task within a package

$ hw airbrake [api-key]
$ hw airbrake 1234567890

Adding a custom package source

To add a custom package source you must provide a name and either the URL of the git repository or the local folder path.

$ hw add_source [name] [source]
$ hw add_source private git@github.com:carrot/hw-packages-private.git
$ hw add_source personal ~/hw-personal-packages

With this approach, you'll have the ability to override existing packages. Note: When overriding packages, the most recently added sources hold greater priority.

After adding a source, you must update all package sources.

Creating a Package

Hemingway is built on top of Thor and thus you have access to all of Thor's actions. Additionally, you have access to additional actions added by Hemingway.

Step 1: Create a file in your source directory/repository with the same name as your class.

$ touch airbrake.rb

Step 2: Encapsulate your class within module HW::Packages.

module HW::Packages
  class Airbrake < HW::Thor
    desc "install ", "Installs Airbrake into the current project"
    def install api_key = false
      unless api_key
        error "You must provide an API key"
        exit(1)
      end

      header "Adding the Airbrake gem to the Gemfile"
      gem "airbrake"

      bundle

      header "Generating Airbrake"
      generate "airbrake --api-key #{api_key}"

      success "Airbrake has been successfully added to your project."
    end
  end
end

Step 3: Run your new package.

$ hw airbrake install abc123xyz789

To view all tasks available in a package, simply run the following:

$ hw airbrake
